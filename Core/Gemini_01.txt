1. Análisis del código fuente
startup_stm32f446retx.s (Archivo de Arranque / Startup)
Este archivo está escrito en lenguaje ensamblador y es lo primero que se ejecuta cuando el microcontrolador recibe energía o se reinicia. Sus funciones principales son:
* Definición de la Tabla de Vectores: Contiene las direcciones de memoria de todas las rutinas de interrupción (ISRs) y excepciones (como Reset_Handler, SysTick_Handler, etc.).
* Reset_Handler: Es el punto de entrada real. Cuando ocurre un reset:
   1. Establece el puntero de pila (sp).
   2. Llama a SystemInit (una función de la librería de ST que configura el sistema de reloj básico, usualmente al oscilador interno).
   3. Copia las variables inicializadas (sección .data) desde la memoria Flash a la memoria RAM.
   4. Inicializa en cero las variables no inicializadas (sección .bss).
   5. Llama a los constructores estáticos (__libc_init_array) si se usa C++.
   6. Finalmente, hace un salto (branch) a la función main() en tu archivo main.c.
main.c (Programa Principal)
Es el núcleo de tu aplicación. El flujo de la función main() es el siguiente:
* HAL_Init(): Inicializa la capa de abstracción de hardware (HAL). Resetea los periféricos y configura el SysTick para generar una interrupción cada 1 milisegundo.
* SystemClock_Config(): Configura el árbol de relojes del microcontrolador (osciladores, PLL y divisores de los buses).
* MX_GPIO_Init() y MX_USART2_UART_Init(): Inicializan los pines de entrada/salida (incluyendo el pin LD2 del LED y el botón B1) y el puerto serie USART2 a 115200 baudios.
* Bucle infinito (while (1)): Alterna el estado del pin LD2 (encendiendo y apagando un LED) e invoca HAL_Delay(500) para esperar 500 milisegundos entre cada cambio.
stm32f4xx_it.c (Rutinas de Interrupción)
Este archivo contiene las funciones que se ejecutan automáticamente cuando ocurre una interrupción de hardware.
* La función más importante aquí para el funcionamiento de main.c es SysTick_Handler(void). Esta función se ejecuta cada vez que el temporizador SysTick llega a cero (configurado para que pase cada 1 ms) y llama a HAL_IncTick(), lo cual incrementa una variable global que el microcontrolador usa como base de tiempo para funciones como HAL_Delay().

2. Evolución de las frecuencias (SystemCoreClock y SysTick)
El sistema pasa por diferentes etapas de frecuencia desde el reset hasta el bucle principal. Aquí tienes la evolución detallada:
Etapa 1: Reinicio y SystemInit (en el archivo Startup)
* SystemCoreClock: El microcontrolador arranca utilizando su oscilador interno RC (HSI - High Speed Internal). En la familia STM32F4, este reloj es de 16 MHz. Por lo tanto, SystemCoreClock es de 16 MHz.
* SysTick: Aún no ha sido inicializado ni configurado.
Etapa 2: Llamada a HAL_Init() (al inicio de main.c)
* SystemCoreClock: Sigue siendo de 16 MHz.
* SysTick: La función HAL_Init() configura el SysTick para que genere interrupciones cada 1 milisegundo (1 kHz). Como el reloj base es de 16 MHz, el registro de recarga del SysTick se configura para contar 16,000 ciclos de reloj antes de disparar la interrupción.
Etapa 3: Llamada a SystemClock_Config() (en main.c)
Aquí ocurre el cambio principal. La función enciende el PLL (Phase-Locked Loop) utilizando el HSI (16 MHz) como fuente para multiplicar la frecuencia.
* Cálculo del reloj: * La señal HSI de 16 MHz se divide por PLLM = 16, resultando en 1 MHz.
   * Ese 1 MHz se multiplica por PLLN = 336, resultando en una frecuencia VCO de 336 MHz.
   * Finalmente, se divide por PLLP = 4 (RCC_PLLP_DIV4), resultando en una salida del PLL de 84 MHz.
* SystemCoreClock: El divisor del bus principal (AHB) se establece en 1 (RCC_SYSCLK_DIV1), por lo que el SystemCoreClock salta de 16 MHz a 84 MHz.
* SysTick: La función interna de ST que aplica esta configuración (HAL_RCC_ClockConfig) detecta el cambio en la velocidad del núcleo y reconfigura automáticamente el SysTick. Ahora, para mantener la interrupción cada 1 ms (1 kHz), el SysTick debe contar 84,000 ciclos de reloj (basado en los nuevos 84 MHz).
Etapa 4: Bucle principal while (1)
* SystemCoreClock: Se estabiliza de forma definitiva en 84 MHz.
* SysTick: Continúa operando a una frecuencia de interrupción de 1 kHz (cada 1 ms), utilizando el reloj central de 84 MHz como base, lo que permite que el HAL_Delay(500) espere exactamente medio segundo.
